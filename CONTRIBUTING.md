# Contributing to ClassCharts Improver

First of all, thank you for taking the time to contribute! Contributions from the community help make this extension a better tool for everyone using ClassCharts.

## üõ†Ô∏è How Can I Contribute?

### Reporting Bugs
* Check the [Issues](https://github.com/jamestheakston1/classchartsimprover/issues) tab to see if the bug has already been reported.
* If not, open a new issue. Include a clear title, steps to reproduce the bug, and your browser version.

### Suggesting Enhancements
* Open an issue with the "enhancement" label.
* Describe the feature you would like to see and why it would be useful for ClassCharts users.

### Pull Requests
1.  **Fork** the repository and create your branch from `main`.
2.  **Develop** your changes. Please adhere to the [Coding Standards](#coding-standards) below.
3.  **Test** your changes locally by loading the unpacked extension in your browser.
4.  **Submit** a Pull Request with a detailed description of your changes.

---

## üíª Coding Standards

To maintain consistency across the project, please follow these requirements when contributing code:

* **Language:** Use **HTML** for structure and layouts wherever possible.
* **Styling:** Use **Tailwind CSS** for styling components.
* **Icons:** Use **Feather Icons** only. Do not use Tailwind Icons or other icon libraries.
* **Privacy:** All data processing must remain **client-side**. Do not write code that transmits user data (profile pictures, goals, or notes) to external servers.

---

## üé® Design Guidelines

When adding UI elements:
* Ensure the design feels like a natural extension of the ClassCharts interface.
* Use Tailwind utility classes to maintain a responsive and modern look.
* Prioritize performance to ensure the extension remains lightweight.

Thank you for your contributions!
